### Technology Stack

| Layer | Technology | Purpose |
|-------|-----------|---------|
| **Frontend** | React + Ant Design | Web/desktop UI |
| **Desktop** | Electron | Cross-platform desktop app |
| **Backend** | FastAPI (Python 3.9+) | REST API, business logic |
| **ORM** | SQLAlchemy | Database abstraction |
| **Database** | SQLite / PostgreSQL | Data persistence |
| **File Storage** | Local / S3 / MinIO | File persistence |
| **Task Queue** | Celery + Redis | Async tasks |
| **Auth** | JWT + RBAC | Security |
| **Caching** | Redis | Performance |
| **Containerization** | Docker + Compose | Deployment |

### Guiding Principles

1. **Modularity** â€” Each core function is a separate module with clean interfaces, enabling independent development and replacement.
2. **Scalability** â€” From single-user local tool to multi-user cloud platform without architectural rewrites.
3. **Configuration over Code** â€” Features driven by configuration files for easy environment adaptation.
4. **Security First** â€” Access control, data encryption (in transit and at rest), and audit logging built in.
5. **Offline by Default** â€” Full functionality without network dependency.

### Deployment Phases

#### Phase 1 â€” Local-First (Current)

- FastAPI backend + SQLite + local file storage on a single machine.
- Zero cost, full data privacy, no internet required.
- Ideal for individual researchers.

#### Phase 2 â€” Hybrid Mode

- Local app as primary interface + optional sync to central cloud backend.
- PostgreSQL + S3/MinIO storage for shared data.
- Ideal for small teams or labs needing controlled sharing.

#### Phase 3 â€” Full Cloud

- Entire application hosted in the cloud.
- Managed database (e.g., RDS), object storage (e.g., S3).
- Web browser + lightweight desktop client.
- Ideal for large organizations or cross-institutional collaboration.

### Reasoning Engine

The **Reasoning Engine** is LabFlow's core automation feature â€” a system for creating, executing, and managing automated analysis workflows as **Directed Acyclic Graphs (DAGs)**.

#### Node Types

| Node Type | Description |
|-----------|-------------|
| `DATA_INPUT` | Selects a file or dataset for processing |
| `TRANSFORM` | Data manipulation (unit conversion, smoothing) |
| `CALCULATE` | Executes a Python script for analysis |
| `CONDITION` | Creates branches based on output values |
| `OUTPUT` | Saves results, generates plots, writes conclusions |

#### Execution Model

1. The engine performs a **topological sort** of nodes.
2. Nodes execute in dependency order, passing results to children.
3. **Fault-tolerant**: error handling and result caching prevent re-computation.
4. Supports **parallel execution** for independent branches.

### Security Architecture

- **Authentication**: JWT tokens with configurable expiry
- **Authorization**: RBAC with three roles â€” Admin, Editor, Viewer
- **Encryption**: AES-256 for data at rest; TLS for data in transit
- **Audit Logging**: Full record of who did what and when
- **File Deduplication**: SHA-256 hashing prevents duplicate storage
