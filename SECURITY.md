# Security Policy

## Public Open-Source Project

- This is a **PUBLIC open-source repository** under GPL v3
- Contributions from the community are welcome
- Please follow the CONTRIBUTING.md guidelines

## Reporting Security Vulnerabilities

If you discover a security vulnerability in LabFlow, please report it responsibly:

1. **Do NOT** open a public GitHub issue
2. Email: [@tico-Lin](https://github.com/tico-Lin)
3. Include: Description, impact, affected versions, and proposed fixes
4. Allow time for a patch before public disclosure

## Code Attribution & IP Protection

- All commits are timestamped and attributed via GitHub
- Full version history is maintained
- Contributors are properly credited
- Modifications are tracked and attributable

---

## Plagiarism Detection

This repository employs the following anti-plagiarism strategies:

1. **GitHub Version Control**: Timestamped, cryptographic commit history
2. **Copyright Notice**: Clear ownership and licensing statements
3. **Legal Documentation**: Comprehensive THIRD_PARTY_NOTICES.md and COPYRIGHT.md
4. **Metadata Tracking**: Git author information and contribution history

### If You Suspect Plagiarism

If you believe your work or this project's code has been plagiarized:

1. Document the evidence (URLs, timestamps, code samples)
2. Note any missing attribution or license notices
3. Report to [@tico-Lin](https://github.com/tico-Lin) with details
